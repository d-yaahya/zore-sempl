وثيقة تخطيط مبدئية — ZORE

منصة زوري لتفصيل العبايات بالذكاء الاصطناعي وإدارة الطلبات والشركاء

هذه الصفحة مرجع مبسط لفكرة المشروع قبل البدء في البرمجة. الهدف بناء منصة متكاملة تسمح للعميل بتصميم عباية حسب اختياراته، توليد تصور بصري بالذكاء الاصطناعي، إرسال الطلب للإدارة أو الخياط، وإدارة كل شيء من لوحة تحكم قوية قابلة للتوسع لاحقًا للمتاجر، المصانع، المصممات، والمسوقات بالعمولة.

01

الرؤية العامة للمشروع

منصة تصميم

العميل يبدأ التصميم خطوة بخطوة: القصة، اللون، القماش، الأكمام، الياقة، التطريز، والوصف النهائي.

ذكاء اصطناعي

تحويل اختيارات العميل إلى وصف ذكي يتم إرساله للذكاء الاصطناعي لتوليد صورة العباية.

نظام تجاري قابل للتوسع

يدعم لاحقًا المتاجر، المصانع، المصممات، المسوقات، الاشتراكات، والإعلانات داخل المنصة.

02

أنواع الحسابات داخل المنصة

العميل

  • تسجيل دخول بالبريد وكلمة المرور.
  • بدء تصميم عباية جديدة.
  • عرض التصاميم السابقة مع التاريخ والصورة والوصف.
  • إمكانية جعل التصميم عامًا في معرض العملاء أو مخفيًا.

الإدارة

  • لوحة تحكم كاملة.
  • مستويات صلاحيات متعددة.
  • إضافة مدراء بدعوة عبر البريد.
  • إدارة الطلبات والمحتوى والتسويق والإعدادات.

المبيعات والموظفون

  • متابعة الطلبات.
  • تغيير حالة الطلب.
  • تأكيد الدفع.
  • تحديث حالة التسليم.

المسوقات بالعمولة

  • رابط تسويقي خاص.
  • احتساب العمولة عند إتمام الطلب.
  • لوحة متابعة للزيارات والطلبات والعمولات.

المصممات والمبدعات

  • حسابات شخصية.
  • ملف أعمال.
  • إمكانية عرض التصاميم.
  • إضافتها لاحقًا كسوق داخلي.

المتاجر والمصانع

  • لوحة تحكم خاصة.
  • هوية مستقلة تظهر للعميل بدل زوري.
  • إدارة الخيارات والأسعار.
  • نظام اشتراك شهري أو سنوي.
03

6التطريزاختيار نوع ومكان وحجم التطريز مع وصف أو صورة مرجعية.
7الوصف النهائيتحويل كل الخيارات إلى نص قابل للتعديل من العميل قبل التوليد.

10إتمام الطلبعرض السعر التقريبي ثم إرسال الطلب للخياط أو فريق المبيعات.

12الأعمال السابقةحفظ التصميم في حساب العميل مع التاريخ والصورة والوصف.
04

رصيد التوليد الأسبوعي

لكل عميل رصيد أسبوعي: 3 تصميمات بالذكاء الاصطناعي، ولكل تصميم 3 تعديلات. يتجدد الرصيد تلقائيًا كل أسبوع. ويمكن لاحقًا إضافة باقات مدفوعة أو اشتراكات تمنح رصيدًا أكبر.

الرصيد المجاني

  • 3 تصميمات AI أسبوعيًا.
  • 3 تعديلات لكل تصميم.
  • عداد واضح داخل حساب العميل.

التطوير المستقبلي

  • شراء رصيد إضافي.
  • باقات VIP.
  • رصيد خاص للمتاجر والمصانع.

07

لوحة التحكم الإدارية

إدارة الموقع

تعديل البنرات، النصوص، الروابط، الناف بار، الفوتر، زر العودة للمتجر، وإظهار أو إخفاء واتساب.

إدارة الخيارات

إضافة القصات، الألوان، الأقمشة، الأكمام، الياقات، التطريزات، الصور، الأسعار، والتوفر.

إدارة الطلبات

مشاهدة الطلبات، الصور، المواصفات، حالة الدفع، حالة التنفيذ، التوصيل، وملاحظات المبيعات.

المدراء والصلاحيات

إضافة مدراء، إرسال دعوات بريدية، تحديد صلاحيات حسب الدور والمهام.

التسويق

إرسال دعوات تجربة الموقع، حملات بريدية، روابط مسوقات، متابعة التحويلات والعمولات.

التحليلات

زيارات الموقع، أكثر القصات اختيارًا، أكثر الألوان طلبًا، معدل إكمال التصميم، الطلبات، والمبيعات.

08

الشركاء والاشتراكات المستقبلية

المتاجر الإلكترونية

  • ربط مع متاجر مثل زد وسلة لاحقًا.
  • رابط خاص لكل متجر.
  • ظهور هوية المتجر بدل زوري للعميل.
  • لوحة تحكم للتاجر لتعديل الخيارات والأسعار.
  • إرسال العمليات إلى بريد التاجر.

المصانع

  • حساب مصنع باشتراك.
  • إضافة الأقمشة والخيارات والأسعار.
  • استقبال طلبات العملاء.
  • عرض أعمال المصنع أو شراء إعلان داخل الموقع.

الرئيسية بدء التصميم معرض أعمالنا أعمال العملاء حسابي طلباتي من نحن المدونة سياسة الخصوصية الشروط والأحكام سياسة الاستخدام العودة للمتجر zore-ksa.com
10

التقنية المقترحة

Frontend

React + Vite لبناء واجهة تفاعلية مناسبة لمراحل التصميم ولوحة التحكم.

Backend

Node.js + Express لبناء API منظم وقابل للتوسع.

Database

MongoDB لتخزين المستخدمين، الطلبات، الخيارات، التصاميم، الشركاء، والإعدادات.

Authentication

JWT مع صلاحيات Roles للعميل والإدارة والموظفين والشركاء.

AI

ربط OpenAI لتوليد الصور من قالب Prompt قابل للتعديل من لوحة التحكم.

Hosting

رفع الكود على GitHub ثم نشر المشروع على Hostinger Node.js مع قاعدة MongoDB خارجية.
